# 참고 프로젝트 제안서
---
## 1. 프로젝트 개요

**프로젝트명:** 당뇨병 위험도 예측 웹 서비스  

본 프로젝트는 사용자가 건강 정보를 입력하면 머신러닝 모델을 통해 당뇨병 위험도를 예측하고, 그 결과를 웹 화면에서 직관적으로 보여주는 서비스를 개발하는 것을 목표로 한다.  

입력 데이터로는 혈당, 혈압, BMI, 나이 등의 건강 정보를 사용하며, 예측 결과는 위험도 퍼센트와 위험 수준 형태로 제공한다. 이를 통해 의료 데이터를 활용한 인공지능 예측 시스템을 이해하고, 웹 서비스 형태로 구현하는 것을 목표로 한다.

---

## 2. 선택 이유

당뇨병은 대표적인 만성질환 중 하나로, 조기 예측과 관리가 매우 중요하다. 또한 공개 데이터셋이 잘 마련되어 있어 머신러닝 프로젝트로 시작하기에 적합하다.  

본 프로젝트를 선택한 이유는 다음과 같다.

- 의료 및 생명과학 분야와 연관성이 높음  
- 공개 데이터셋을 활용하여 비교적 쉽게 구현 가능  
- 머신러닝 모델을 웹 서비스 형태로 확장할 수 있음  
- 데이터 분석, 모델링, 웹 개발을 함께 경험 가능  

---

## 3. 프로젝트 분석 내용

### 3-1. 데이터셋

https://github.com/Aditya-Mankar/Diabetes-Prediction

본 프로젝트에서는 **Pima Indians Diabetes Dataset**을 사용한다.  

이 데이터셋은 다음과 같은 건강 정보를 포함한다.

- Pregnancies (임신 횟수)  
- Glucose (혈당)  
- BloodPressure (혈압)  
- SkinThickness (피부 두께)  
- Insulin (인슐린 수치)  
- BMI (체질량지수)  
- DiabetesPedigreeFunction (유전적 위험도)  
- Age (나이)  

출력값은 당뇨병 여부(0 또는 1)이다.

### 3-3 . 프로젝트 분석

#### 데이터셋
768개 샘플과 8개 입력 변수, 1개 타깃으로 구성되며, train/test 8:2 분할을 사용했다. 최종 모델은 선형 SVM이며 MinMax scaling을 적용했다.

#### 모델 
sklearn.svm.SVC의 선형 커널 SVM이다. 학습 후 model.pkl로 저장하고, Flask 앱이 이 파일을 로드해서 예측하는 구조다
데이터셋은 표 형식의 중소형 데이터셋이고, 이미지나 시계열이 아니라서 CNN이나 RNN보다 전통적인 분류기가 더 자연스럽다. 특히 feature 수가 많지 않고 2진 분류 문제이기 때문에 SVM은 꽤 전형적인 선택이다. 게다가 프로젝트에서 데이터 값을 0과 1 사이의 범위 값으로 변환하는 MinMaxScaler로 입력값을 0~1로 정규화했기 때문에, 스케일에 민감한 SVM과도 잘 맞는 편이다

#### 데이터 전처리

결측치(0값) 처리: 포도당, 혈압, BMI 등에서 발견된 논리적 오류(0)를 평균값 또는 중앙값으로 대체하여 데이터의 손실을 방지함.
