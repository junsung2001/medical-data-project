## 1. 중간 결과 및 데이터 분석

### 1-1. 데이터셋 수집 및 전처리 성과
* **활용 데이터:** Pima Indians Diabetes Dataset (총 768행, 9개 피처)
* **결측치 및 논리 오류 정제:** 의학적으로 '0'이 될 수 없는 주요 지표(`Glucose`, `BloodPressure`, `SkinThickness`, `Insulin`, `BMI`)의 오류 데이터를 감지하고, 비정상적인 결측치들을 이상치에 강한 **중앙값(Median)**으로 대체하여 데이터의 신뢰성을 확보함.
* **스케일링:** 거리 기반 알고리즘인 SVM의 특성을 고려하여, 모든 수치형 피처를 0과 1 사이로 정규화하는 `MinMaxScaler`를 적용함.

### 1-2. 머신러닝 모델 학습 결과 및 검증
참고 프로젝트(`reference_analysis.md`)의 한계를 극복하기 위해 **2단계에 걸친 모델 고도화**를 수행했습니다.

* **1차 시도 (Linear SVM):** 전체 정확도는 양호했으나 당뇨 환자(1.0)를 잡아내는 **재현율(Recall)이 0.48**에 그쳐, 실제 환자를 절반 이상 놓치는 치명적인 의료 오진 위험성이 발견됨.
* **2차 고도화 시도 (RBF Kernel + Class Weight):** 데이터셋의 클래스 불균형(정상군 대비 환자군 부족)을 해결하기 위해 `class_weight='balanced'`를 적용하고 비선형 패턴을 학습하는 RBF 커널로 전환함.

## 2. 수정 및 구체화 목표 (피드백 반영)

### 2-1. 클라우드 배포 시 비용 및 안정성 방어 목표 추가
수정 목표: 오픈소스 모니터링 및 사용자별 API 토큰 쿼터(Quota) 제한 기능 내장.

이유: Google Cloud Platform(GCP) 환경에 챗봇 서버를 배포할 때, 무제한적인 LLM API 호출로 인한 비용 폭탄(요금 리스크)을 선제적으로 차단하기 위함입니다. 
이를 위해 앱 레벨에서 유저별 하루 제한 한도를 걸고, 오픈소스 모니터링 툴(Prometheus, Grafana) 및 Langfuse를 결합하여 트래픽과 누적 비용 동향을 대시보드로 실시간 추적하는 운영 목표를 추가했습니다.

### 2-2. 정밀도(Precision) 저하에 따른 차별화 전략 강화

문제점 및 고려 사항: 환자를 잡기 위해 모델의 민감도(Recall)를 높인 결과, 정상인을 위험군으로 오탐지하는 비율(False Positive)이 일부 증가함.

수정 및 보완 목표: 오탐지된 사용자에게 올바른 피드백을 전달하여 사용자 경험(UX)이 저해되지 않도록, 당초 단순 안내 방식에서 '예측 확률 연동형 AI 건강 상담 챗봇'으로 차별화 기능을 강화함.

이유: 예측 결과인 '위험도 %'와 사용자의 건강 수치를 LLM(대화형 AI)의 시스템 프롬프트에 동적으로 결합하여, 사용자가 오진을 받더라도 불안해하지 않고 일상 속에서 예방 조치를 취할 수 있도록 양방향 가이드를 제공하기 위함입니다.
